# Jtool JSON 工具测试指南

### 1. 安装依赖

```bash
cd /home/admin/.openclaw/workspace/jtool-fix
```

## ✅ 测试清单

### 任务一：功能修复验证

#### 1.1 转义功能测试

**测试用例 1：字符串转义**
```json
// 输入 1（纯文本）
Hello "World" with special chars: \n \t

// 预期输出
Hello \"World\" with special chars: \\n \\t
```

```json
// 输入 2（JSON 对象）
{"name": "John", "message": "Hello\nWorld"}

// 预期输出
{\"name\": \"John\", \"message\": \"Hello\\nWorld\"}
```

**测试用例 2：去除转义**
```json
// 输入
Hello \"World\" with \\n \\t

// 预期输出
Hello "World" with \n \t
```

**测试用例 3：全部转义**
```json
// 输入
{"key": "value"}

// 预期输出
\{\”key\”:\ \”value\"\}
```

**测试用例 4：全部去转义**
```json
// 输入
\{\”key\”:\ \”value\"\}

// 预期输出
{"key": "value"}
```

#### 1.2 格式化功能测试

**测试用例：格式化压缩 JSON**
```json
// 输入
{"name":"John","age":30,"city":"New York","hobbies":["reading","gaming"]}

// 预期输出
{
  "name": "John",
  "age": 30,
  "city": "New York",
  "hobbies": [
    "reading",
    "gaming"
  ]
}
```

#### 1.3 压缩功能测试

```json
// 输入
{
  "name": "John",
  "age": 30
}

// 预期输出
{"name":"John","age":30}
```

#### 1.4 校验功能测试

**测试用例 1：有效 JSON**
```json
{"valid": true}
// 预期：显示 ✅ JSON 格式正确
```

**测试用例 2：无效 JSON**
```json
{"invalid": true,}
// 预期：显示 ❌ 错误信息
```

#### 1.5 错误信息显示测试

- [ ] 输入无效 JSON 时显示红色错误框
- [ ] 错误信息包含具体错误原因
- [ ] 可以手动关闭错误提示
- [ ] 执行新操作时自动清空之前的错误
